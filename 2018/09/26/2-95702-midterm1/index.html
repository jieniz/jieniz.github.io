95-702 Distributed System Midterm 1

Posted on 2018-09-26 | In 95-702 Distributed System |

Introduction

Characterization of DS

  • 同步：Components are located on networked computers and execute concurrently.
  • 独立：Components communicate and coordinate only by passing messages.
  • 异时：Time differs on each system.

Four styles of application integration

How does program logic share data or communicate?

  • 文件共享：Shared files
  • 数据库共享：Shared database
  • 远程调用：Procedure Call or Method Invocation (RPC, RMI)
  • 第三方：Messaging through a third party

Synchronous and asynchronous communication

  • 同步：client发消息, 等待, 限定各种bound
  • 异步：client发消息, 然后继续放飞，不限定任何bound

Communication Paradigms

  • Interprocess:
    • low level
    • coupled in time
    • TCP / UDP / Multicast Sockets
  • Remote: Two way exchange with a remote operation, procedure or method
    • Higher level abstractions
    • coupled in time / space
    • RPC: Remote procedure calls, call procedures in processes on remote computers
    • RMI: Remote method invocation, invoke a method in a remote object
    • HTTP, DCOM, CORBA
  • Indirect communication: less tightly coupled and involving a third party
    • Publish-subscribe (AKA distributed event based systems): routes messages to interested parties. One-to-many style of communication.
    • Message queues (AKA channels): for point-to-point messaging.
    • Tuple spaces: allows for the placement and withdrawal of structured sequences of data.

Time and space coupling

Coupling is the degree to which some communicating entity makes assumptions about its partner.

  • Coupled in time: 双方同时存在
  • Coupled in space: 知道对方是谁

Layered architecture

vertical: the separation of concerns found in a model, view, controller design pattern.

  • applications and services layered on the top.
  • middleware appears between the application and the operating system.
  • The operating system sits on top of the computer and network hardware.

Tiered architecture

complimentary to layering

  • 应用层：usually applied to the applications and services layer (client-server)
  • 水平分层：organize the functionality of a given layer and place this functionality into appropriate servers and onto physical devices
  • include: presentation logic, business logic, and data logic
    • client: business logic and user interface
    • server: data logic layer
  • partition: application into two tiers or three.
    • 2-tier: client-server architecture design patterns
      • request/response: 一来一回
        • /client/ -> request -> /server/ -> response -> /client/
      • request/acknowledge: 一来一回，server认证
        • /client/ -> request -> /server/ anthentication -> acknowledgement -> /client/
      • request/acknowledge/poll: 两次请求，第一次放入数据库，第二次拉回
        • /client/ -> request -> /service1/ -> save ->/database/ -> acknowledge -> /client/
        • /client/ -> poll request-> /service2/ -> query for data -> /database/ -> data -> /service2/ -> response -> /client/
      • request/acknowledge/callback: 一次请求，server后台与callback交互
        • /client/ -> request -> /service/ -> process -> /callback processor/
        • /service/ -> acknowledge -> /client/
    • 3-tier: logic layer correspond directly to the physical machines and processes
      • AJAX application: google maps are fetched as needed
      • thin client approach: Move complexity into internet based services (cloud computing)
  • Main driver: promote separation of concerns.

Software engineering principle: Separation of concerns

tiered -> prmote seperation of concern -> raise the level of abstraction

Architectural patterns

Proxy pattern

the client makes calls on a local object (the proxy) that has the same interface as a remote object. The proxy hides the communication details.

Brokerage pattern

consists of a trio of service provider, service requestor and service broker (typically with lookup and bind operations)

Challenges in constructing DS

All the challenges associated with stand alone systems development plus:(Pony Express)

  • Heterogeneity: hinders interoperability 异质阻碍协同 (geography)
  • Security (robber)
  • Scalability / bandwidth：在增加大量资源和用户的情况下，系统的效率仍能保持不变 (letter/horse)
  • Failure handling: (networks, processors, remote systems) Independent Failure of components
  • Concurrency: adds complexity
  • Openness: easy to add to or modify the system

Pepperland

Agreement

将交流双方的latency控制在（max - min + 1）内。
A 和 B 准备一起打 C ，但是 A B 隔太远，又没有电话，只能靠人马。就约定好，A 发出攻击指令后，等待 MIN ，然后就出发；而 B 收到攻击指令后，等待 1 秒后出发。

Failure Detection

双方无法同时知道交流失败了。

Networking and Internetworking

Application Protocols

进程间为了交流而制定的一系列规则: (1)消息序列中的某一部分一定被交换了, (2)消息含有特定格式的数据

HTTP

Browser to server, web app to web service, etc.

DHCP

Dynamic Host Configuration Protocol：server assigns IP address to a host 动态的在可用的子网区间内获取host的IP地址

ARP

Address Resolution Protocol: 在本地网络转换 host 的 IP 地址和 mac 地址

  • 每个 host 都有 ARP 模块, 存储 mac / IP address pairs
  • 已知 H2 IP address, host H1 想连接 H2，H1 查缓存，看是否存在pair
    • yes: 直接用 H2 mac address
    • no: broadcast IP address, H2 match IP 返回自己的 mac address, H1 把mapping加入缓存

Ports and Port Assignment

  • 端口: 信息传送时主机的一个终点, 指定信息发送到某个目标节点的特定程序。
  • 分配：
    • 0 to 1023: Well Known Ports
      • Should not be used without IANA registration
      • Loosely, system-type-services ports
      • 80: HTTP port
    • 1024 to 49151: Registered Ports
      • Should not be used without IANA registration
      • Loosely, application-level ports
    • 49152 to 65535: Dynamic and/or Private Ports – Anything

Addressing

  • IP: 网络中的每个主机都分配了一个 IP 号码，可以用来识别特定主机以及其连接的子网。
  • MAC: 每个及其都有一个唯一的 MAC 地址，由厂家分配。

Datagram

包交换网络中传输的最小单位

  • one-shot 过程：不用任何设置，一旦发送了就不管了。所以是 unreliable。
  • IP datagram: contains the IP address of the destination host.

Protocol Layers

由下到上，由内向外

Application Layer

HTTP request / response

  • abstraction:
  • addressing: GET /hi.html HTTP/1.1 or HTTP/1.1 200 OK

Transport Layer

TCP / UDP socket

  • abstraction:
  • addressing: port

TCP / UDP Compared

  • TCP：面向连接的协议。打电话，一头连另一头。速度慢。有顺序。有很多头部信息。可靠。
  • UDP：面向数据的协议，不管连接。寄信一样，写了地址扔邮筒我们就无法掌控它了。速度快。没顺序。没啥头要包起来的。不可靠，容易丢包。

Internet Layer

IP协议

  • abstraction: Single IP Datagram from one Internet host to another Internet host
  • addressing: IP address

Data Link Layer

Ethernet

  • abstraction: Single frame from one host to another host ON THE SAME NETWORK.
  • addressing: MAC address

RIP algorithm

Routing: 两点之间的路径，路由器里存储路径表，包含了网络中本地到网络中所有节点的距离（成本）
RIP / Routing Information Protocol: 用于更新路由器的路径表的简单算法

  • 每当自己的路径表改变了，就把表发送到所有邻居。
  • 当从附近路由器收到一张表，如果出现了一个新终点，或者出现了比现有成本更小的路径，则更新表。
