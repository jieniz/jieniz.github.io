T(N)法分析时间复杂度

Posted on 2018-11-06 | In Leetcode |

树形分析法：逐层分析，最后累加

T(N) = T(N/2) + O(1)

T(N) = T(N/2) + O(1) = T(N/4) + 2O(1) = T(N/8) + 3O(1) = … = T(1) + logN*O(1) = O(logN)

  • 解释：把规模为N的问题(T(N))分解为T(N/2)需要花费O(1)的时间，把T(N/2)分解为T(N/4)需要花费O(1)，把T(N/4)分解为T(N/8)需要花费O(1)的时间……，最后分解到T(1)需要O(1)的时间，每层需要O(1)时间，共logN层，累加得到时间复杂度为O(logN)
  • 例子：binary search (array or BST)

T(N) = T(N/2) + O(N)

T(N) = T(N/2) + O(N) = T(N/4) + (O(N)+O(N/2)) = T(N/8) + (O(N)+O(N/2)+O(N/4)) = … = T(1) + O(N+N/2+N/4+…+1) = O(N)

  • 解释：把T(N)分解为T(N/2)需要花费O(1)的时间，把T(N/2)分解为T(N/4)需要花费O(N/2)的时间，把T(N/4)分解为T(N/8)需要花费O(N/4)的时间，……，最后分解到T(1)需要花费O(1)的时间，每层时间累加N+N/2+N/4+…+1最后得到时间复杂度为O(N)

T(N) = 2T(N/2) + O(1)

T(N) = 2T(N/2) + O(1) = 4T(N/4) + (O(1)+2O(1)) = 8T(N/8) + (O(1)+2O(1)+4*O(1)) = … = O(1+2+4+8+…+N) = O(2N-1) = O(N)

  • 解释：把T(N)分解为T(N/2)需要花费O(1)的时间，但两个T(N/2)都需要遍历，因此系数也要不断累加，每一层的时间分别为O(1), O(2), O(4), …, O(N)，共logN层，最后把每一层的时间加起来
  • 例子：binary search with duplicates

T(N) = 2T(N/2) + O(N)

T(N) = 2T(N/2) + O(N) = 4T(N/4) + (O(N)+2O(N/2)) = 8T(N/8) + (O(N)+2O(N/2)+4O(N/4)) = … = NT(1) + O(N+N+N+…+N) = O(NlogN)

  • 解释：把T(N)分解为T(N/2)需要花费O(N)的时间，把2个T(N/2)分解为T(N/4)需要花费2个O(N/2)的时间，把4个T(N/4)分解为T(N/8)需要花费4个O(N/4)的时间，……，最后分解到T(1)需要花费N个O(1)的时间，每层时间均为O(N)，共logN层，最后得到时间复杂度为O(NlogN)
  • 例子: merge sort, quick sort
